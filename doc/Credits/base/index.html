OpendTect - Credits

Introduction

OpendTect development started back in 1999 when StatoilHydro (then Statoil) requested dGB to develop a software system around the seismic object detection method the companies had jointly invented for chimney interpretation. It was developed with financial support from StatoilHydro and SenterNovem (The Dutch government).

The project (initially named 'd-Tect') was a conventional package - a closed source system for seismic attribute processing and object detection. In 2003 dGB decided it was time to go Open Source.

To make this happen, parts of d-Tect had to be re-engineered. In a few months, a few closed-source parts were split off to fit in a new plugin-based architecture. d-Tect became OpendTect, the first real Open Source seismic interpretation system. The plugin architecture has been crucial since then, it makes sure OpendTect can be extended at run-time with plugins of any type, whether free or commercial.

dGB sustains development by funding from the industry & governments, maintenance fees collected from commercial users of OpendTect Base (The Open source part), and income from the commercial plugins. Projects

Gaz de France (2005)

Jean-Francois Dutzer of Gaz de France proposed (and funded) various improvments to the Open Source part of OpendTect. To name few:

  • Random lines through wells
  • Create New Well track through picking
  • Move element to top/ to bottom
  • Set gridlines option
  • Display horizon at section only
  • Lock/ Unlock elements in the tree
  • Making Colorbars more user friendly

BG 2008 project

Before rolling out OpendTect + plugins to all E&P users within BG, Hamish Macintyre and Charles Jones of the Advanced Geoscience Group put together a wish-list with general improvments the BG users would like to see implemented in OpendTect Base (the open source part). Substantial funds were raised and between end-2007 and mid-2009 dGB implemented many improvments in versions v3.2 and v3.3. To name few:

  • Various visualization improvments
  • New attributes
  • Pre-stack SEGY access and other SEG-Y improvements
  • Crossplot re-engineering / 3D connection
  • Color blending
  • Color bar / Scaling
  • Zero-offset VSP
  • Pre-stack viewing
