Credits - the GapDecon plugin

History

This plugin was created for Addax Petroleum. The initiator was Bernd Fiebig, who frequently needed post-stack multiple removal capabilities for his work. Being a user of OpendTect already, he found funds to get GapDecon functionality incorporated in OpendTect.

The type of multiple removal algorithm chosen for this application was the well known least squares prediction error filter, a.k.a "Gap Decon". This filter aims to flatten a user defined part of an auto-correlation function.

Implementation

Gap Decon was implemented by Hélène Payraudeau (now: Hélène Huck).
